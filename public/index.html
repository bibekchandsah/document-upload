Project Info & Limitations

Features

  • GitHub-based file storage and management
  • Drag & drop file uploads
  • Folder organization with nested folders
  • Google Docs viewer for document preview
  • Search functionality
  • Renaming files and folders
  • Bulk operations (move, copy, delete)
  • Expiring share links with custom duration
  • QR code generation for share links
  • Progressive Web App (installable)
  • Real-time GitHub API rate limit monitoring

GitHub Limitations

  • Maximum file size: 100MB per file
  • API rate limit: 5,000 requests/hour
  • Repository size limit: ~1GB recommended
  • Low bandwith: >1GB repository size
  • Large files (>100MB) require Git LFS
  • File operations may be slower than traditional storage
  • GitHub token required for authentication
  • Renaming files/folders >1MB takes longer duration

Best Practices

  • Keep files under 50-100MB for best performance
  • Use private repositories for sensitive data
  • Before uploading make sure it's properly named
  • Monitor your API rate limit usage regularly
  • Organize files in folders for easy management
  • Generate time-limited share links for security

About This Project

This is a GitHub-based document management system that leverages GitHub's infrastructure for file storage and version control. Perfect for personal document management, small team collaboration, and learning about API integration.
